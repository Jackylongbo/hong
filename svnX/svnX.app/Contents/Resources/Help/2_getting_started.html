2. Getting started

2.1. If you are new to Subversion

You should definitely get familiar with Subversion before using svnX!
The book is available in HTML or PDF at <svnbook.red-bean.com>.
Also, learn the command-line basics. Anyway, you won’t be able to set up a server or repository with svnX.

2.2. Requirements

SvnX is built as a Universal binary application. It requires the following to run:

  • Mac OS X 10.4 or later.
  • A working install of Subversion 1.4, 1.5 or 1.6.
    The recommended Subversion 1.6.13 installer for Mac OS X 10.5 (Leopard) is available here
    and for Mac OS X 10.6 (Snow Leopard) here.
    The recommended Subversion 1.5.7 installer is available here.
    The recommended Subversion 1.4.6 installer is available here.
In order to benefit from the new accelerated working copy refresh feature Subversion must have been installed in /opt/subversion. Additionally if the above Subversion 1.4.6 installer was used the following should be executed in a terminal window:

    cd /opt/subversion/lib; ln -s libapr-0.0.9.*.dylib libapr-1.0.dylib

If you’re using another package, make sure that the path to svn binaries is correct, in svnX preferences. Initially the path is set to “/usr/local/bin”, “/opt/subversion/bin” or “/usr/bin”.

The first time you launch svnX it will look in the above locations for a copy of the Subversion command line tool ‘svn’. If it can’t find one it will notify you and ask you to locate one.
